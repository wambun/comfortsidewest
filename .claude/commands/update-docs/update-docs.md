# How to update project documentation

Please update the project documentation for: $ARGUMENTS.

Follow this systematic approach:

## Documentation Analysis

1. Review current documentation status:
   - Check `CLAUDE.md` for project overview and current instructions
   - Review `README.md` or template README files for project setup and usage
   - Examine `.cursor/rules/` directory for rule documentation
   - Check `package.json` for dependencies and scripts
   - Review any changelog or release notes files
   - Examine component documentation and examples

2. Analyze recent implementation and changes:
   - Review what was implemented in recent commits
   - Check for new components, features, or functionality
   - Identify new dependencies or configuration changes
   - Note any implementation patterns or best practices discovered
   - Review any testing results or issues encountered
   - Cross-reference documentation with actual implementation

## Documentation Updates

1. Update CLAUDE.md:
   - Add new implementation guidelines discovered during development
   - Update project overview if scope has changed
   - Add new best practices and patterns
   - Document any new tools or workflows
   - Update code quality and verification procedures
   - Add troubleshooting guidance for common issues

2. Update README.md and template files:
   - Update installation and setup instructions
   - Add new features to feature lists
   - Update usage examples with new functionality
   - Document new configuration options
   - Add deployment notes for new features
   - Update frequently asked questions

3. Update Cursor Rules (.cursor/rules/):
   - **nextjs.mdc**: Update with new Next.js patterns and best practices
   - **ui-components.mdc**: Add new Shadcn UI components and usage patterns
   - **tailwind-styling.mdc**: Update styling conventions and new patterns
   - **tech-stack-dependencies.mdc**: Update with new dependencies and versions
   - **project-structure.mdc**: Update with new directories or file organization
   - **landing-components.mdc**: Add new landing page components or patterns
   - **blog-posts.mdc**: Update blog writing guidelines and new features
   - **typescript-style.mdc**: Add new TypeScript patterns and conventions
   - **self-improve.mdc**: Update rule improvement processes
   - **cursor-rules.mdc**: Update rule management guidelines
   - **tools.mdc**: Add new tools or update existing tool usage

4. Update component and feature documentation:
   - Document new React components and their props
   - Add usage examples for new features
   - Update API documentation if applicable
   - Document new page routes or layouts
   - Add configuration examples for new options

5. Update development workflow documentation:
   - Document new build processes or scripts
   - Add testing procedures for new features
   - Update deployment instructions
   - Document new development tools or workflows
   - Add troubleshooting guides for common issues

## Documentation Formatting and Structure

1. Maintain consistent documentation style:
   - Use clear headings and sections
   - Include code examples where helpful
   - Use proper Markdown formatting
   - Maintain consistent naming conventions
   - Include links between related documentation

2. Ensure documentation completeness:
   - Cover all implemented features
   - Include practical usage examples
   - Document configuration options
   - Include troubleshooting guidance
   - Provide clear setup and development instructions

## Guidelines

- UPDATE existing documentation files rather than creating new ones
- Maintain consistent documentation style across all files
- Include practical examples and code snippets
- Cross-reference related documentation sections
- Document best practices and lessons learned
- Ensure documentation reflects actual implementation
- Update version numbers and dependency information
- Include troubleshooting guides for common issues
